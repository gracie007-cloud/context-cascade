# GitHub Project Management

Operate GitHub Projects with governance, automation, and reporting.

## When to Use
- Creating or refining GitHub Projects for delivery/roadmaps
- Automating issue/PR workflows and reporting
- Establishing SLAs, labels, and governance

## When Not to Use
- Release orchestration (use github-release-management)
- Cross-repo code campaigns (use github-multi-repo)
- Simple workflow updates (use github-workflow-automation)

## Workflow Snapshot
1. Clarify objectives, permissions, and reporting needs.
2. Design fields/views and automation with approvals.
3. Run cadences (triage, planning, review) with alerts.
4. Publish dashboards and iteratively improve.

## Artifacts
- SKILL.md SOP, examples/, tests/, resources/, references/, and this readme

## Quality Gates
- Governance/permissions validated
- Automation tested before full rollout
